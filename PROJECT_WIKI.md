## 📋 МОДЕЛЬ ДАНИХ — ШВИДКА РЕФЕРЕНЦІЯ

### Plant Batch (Партія)
```
batch_title      — назва
status           — Живцювання/Укорінення/Зростання/Готова до пересадки/Готова до продажу/Закрита/Втрачена
plant_variety    — link → Item
source_type      — Живцювання/Купівля/Насіння/Поділ куща/Розділення партії
mother_plant     — link → Mother Plant (якщо Живцювання)
parent_batch     — link → Plant Batch (якщо Розділення)
start_date       — дата старту
expected_ready_date
quantity_start   — початкова к-сть
quantity_current — поточна (read-only, рахується)
quantity_lost    — втрати (read-only)
current_location — link → Warehouse
pot_size         — Касета/0.5L/1L/2L/3L/5L/7L/10L/15L/20L/35L
shelf_row        — стелаж/ряд (текст)
total_cost       — грн (auto)
cost_per_plant   — грн/рослина (auto)
expected_loss_rate — % (default 15%)
notes
```

### Batch Event (Подія)
```
event_date
batch            — link → Plant Batch
event_type       — 🧪 Обробка / 💧 Підгодівля / 🌱 Пересадка / 📍 Переміщення / ☠️ Списання / 📝 Спостереження
performed_by     — link → User
treatment_reason — Планова/Реактивна/Екстрена
product_used     — link → Item (препарат/добриво)
quantity_used    — float
unit             — мл/л/г/кг/шт
cost             — грн
quantity_lost    — к-сть загинуло
loss_reason      — Хвороба/Шкідник/Мороз/Посуха/Мех.пошкодження/Не вкорінилось/Інше
notes
next_action_date
```

### Mother Plant (Маточник)
```
plant_variety    — link → Item
```

## 🔄 ВИРОБНИЧИЙ ЦИКЛ (БІЗНЕС-ЛОГІКА)

```
СТАРТ ПАРТІЇ
  ↓ (з маточника або купівля)
ЖИВЦЮВАННЯ → Plant Batch [статус: Укорінення]
  ↓ (4-8 тижнів)
ПОДІЇ: Спостереження, Обробки, Списання загиблих
  ↓
ПЕРЕСАДКА → Repotting Record → нові дочірні Plant Batch
  ↓
ЗРОСТАННЯ → ще події
  ↓
ГОТОВА ДО ПРОДАЖУ → Stock Entry в ERPNext
  ↓
ПРОДАЖ → POS або Sales Invoice
```

**Розрахунок собівартості:**
`cost_per_plant = total_cost / quantity_current`
`total_cost` = сума всіх cost з Batch Events

## 🛠️ ТЕХНІЧНІ НОТАТКИ

### MCP ERPNext
- Endpoint: frappe.client.get_list, frappe.client.insert, frappe.client.get
- Права: Custom DocPerm → role "All" → RWCD
- Naming Plant Batch: `BATCH-.YYYY.-.MM.-` (автонумерація)
- Naming Batch Event: `EVT-.YYYY.-.MM.-`

### Відомі обмеження
- `frappe.permissions.add_permission` — не whitelist-ований, треба Custom DocPerm
- `list_documents` з filters — не працює (помилка API), використовуй `run_method` + frappe.client.get_list
- `quantity_current` та `quantity_lost` — read_only в DocType, треба Server Script для автооновлення

### Виправлення що треба зробити
- [ ] Server Script: при збереженні Batch Event з quantity_lost → оновити Plant Batch.quantity_lost і quantity_current
- [ ] Server Script: при збереженні Batch Event з cost → оновити Plant Batch.total_cost і cost_per_plant
- [ ] Теплиця А — тип "Transit" (треба змінити на null або "Storage")
