# SKILL: Batch Event — Запис подій партії

**Коли використовувати:** "запиши спостереження", "списати", "додай витрати", "туя виглядає добре"

---

## Типи подій

| Тип | Поле event_type | Коли |
|---|---|---|
| 📝 Спостереження | `Спостереження` | нотатка без змін кількості |
| ☠️ Списання | `Списання` | рослини загинули |
| 💊 Обробка | `Обробка` | хімія, фунгіциди |
| 🌱 Підгодівля | `Підгодівля` | добриво |
| 💰 Витрати | `Витрати` | купили матеріали |
| 📦 Пересадка | `Пересадка` | перехід в більший горщик |

---

## Алгоритм запису

### Крок 1 — Уточнити партію
Якщо юзер не вказав партію точно:
```
→ erpnext_run_method: get_list Plant Batch (активні)
→ Показати список, попросити уточнити
```

### Крок 2 — Створити Batch Event
```python
erpnext_create_document("Batch Event", {
    "batch_id": "[ID партії]",      # обов'язково
    "event_type": "[тип]",           # обов'язково  
    "event_date": "[YYYY-MM-DD]",    # сьогодні якщо не вказано
    "notes": "[нотатка]",            # опис
    "quantity_lost": 0,              # якщо списання
    "cost": 0,                       # якщо витрати (грн)
})
```

### Крок 3 — Перевірити що Server Script спрацював
Після insert — перевірити Plant Batch:
- `quantity_current` зменшилось (якщо списання)
- `total_cost` і `cost_per_plant` оновились (якщо витрати)

## Приклади розпізнавання намірів

| Фраза юзера | Тип | Поля |
|---|---|---|
| "туя виглядає добре, 175 шт" | Спостереження | notes |
| "загинуло 5 штук" | Списання | quantity_lost=5 |
| "купили субстрат на 500 грн" | Витрати | cost=500 |
| "обробили Фундазолом" | Обробка | notes |
| "підгодували" | Підгодівля | notes |

---

## Важливо
- Server Script автоматично оновлює Plant Batch після кожного Event
- НЕ оновлювати Plant Batch вручну якщо є Event — Script зробить сам
- Дата події = сьогодні якщо юзер не вказав інше
