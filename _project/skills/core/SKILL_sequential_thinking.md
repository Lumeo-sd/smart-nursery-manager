# SKILL: Sequential Thinking — Покрокове вирішення задач

**Коли використовувати:**
- Складні архітектурні рішення ("як краще організувати...")
- Вибір між кількома підходами
- Діагностика проблем де причина неочевидна
- Будь-що де потрібно "добре подумати перед дією"

---

## Як це працює

`sequential-thinking` — це інструмент що змушує мислити послідовно, крок за кроком, з можливістю переглянути попередні висновки.

Кожна "думка" може:
- Розвивати попередню
- Переглядати і виправляти попередню
- Розгалужуватись в новий напрямок

---

## Коли ОБОВ'ЯЗКОВО застосовувати в нашому проекті

| Ситуація | Чому важливо |
|---|---|
| Реорганізація файлової структури | Ризик щось зламати |
| Зміни в ERPNext схемі (DocTypes) | Можуть бути залежності |
| Вибір архітектури (PWA, API) | Довгострокові наслідки |
| Діагностика помилок Docker/ERPNext | Багато змінних |
| GitHub структура і git flow | Складно відкотити |

---

## Правило

Перед будь-якою незворотною дією (видалення, переміщення файлів, зміна схеми БД) — спочатку sequential-thinking, потім дія.
