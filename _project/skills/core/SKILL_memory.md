# SKILL: Memory MCP — Персистентна пам'ять між сесіями

**Коли використовувати:**
- Запам'ятати важливе рішення між сесіями
- Зберегти контекст який не поміщається в CHAT_CONTEXT.md
- Тимчасові нотатки під час роботи

---

## Що це

`memory` MCP зберігає інформацію у вигляді **графу знань** (entities + relations).
Дані живуть між сесіями — не скидаються при закритті чату.

## Інструменти

| Інструмент | Коли |
|---|---|
| `create_entities` | Зберегти новий факт (клієнт, рослина, рішення) |
| `add_observations` | Додати деталі до існуючого факту |
| `create_relations` | Зв'язати два факти |
| `search_nodes` | Знайти збережене |
| `read_graph` | Переглянути все збережене |
| `delete_entities` | Прибрати застаріле |

---

## Що зберігати для нашого проекту

**Зберігати в memory:**
- Рішення що прийняли і чому (архітектурні вибори)
- Помилки що зустрічались і як вирішили
- Плани на наступну сесію якщо не встигли

**НЕ зберігати в memory (є кращі місця):**
- Стан системи → `CHAT_CONTEXT.md`
- Skills → `_project/skills/`
- Код → файли проекту

---

## Важливо

`CHAT_CONTEXT.md` — головний файл стану. Memory MCP — доповнення для деталей що не варто тримати в головному файлі.
