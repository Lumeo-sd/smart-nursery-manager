# SKILL: Time MCP — Час і часові зони

**Модуль:** `mcp/time/`
**Коли:** потрібна точна дата/час, розрахунок дедлайнів, часові зони

---

## Доступні інструменти

| Інструмент | Аргументи | Що робить |
|---|---|---|
| `get_current_time` | `timezone` (IANA) | Поточний час у вказаній зоні |
| `convert_time` | `source_timezone`, `time` (HH:MM), `target_timezone` | Конвертація між зонами |

**Часова зона проекту:** `Europe/Kyiv`

---

## Використання в проекті розсадника

### Де це потрібно

| Ситуація | Як використовую |
|---|---|
| Дата Batch Event | `get_current_time(Europe/Kyiv)` → беру дату |
| Ранковий дайджест | Показую точний час генерації звіту |
| Розрахунок готовності партії | Поточна дата + тижні укорінення |
| Планування живцювання | Перевірити чи настав оптимальний час |

### Приклад — дата для Batch Event
```
get_current_time("Europe/Kyiv")
→ 2026-03-12T14:30:00+02:00
→ беру: event_date = "2026-03-12"
```

### Приклад — коли готова партія?
```
Партія Туя живці 2026-03-01, укорінення ~10 тижнів
get_current_time("Europe/Kyiv") → сьогодні 2026-03-12
Залишилось: ~8.5 тижнів → очікувана готовність: ~2026-05-10
```

## Важливо

- Завжди використовуй `Europe/Kyiv` (UTC+2/+3 залежно від сезону)
- Для дат ERPNext: формат `YYYY-MM-DD`
- Для відображення: `DD.MM.YYYY` (зручніше для персоналу)
